## Run & Operate

- `pnpm --filter @workspace/api-server run dev` вҖ” run the API server (port 8080)
- `pnpm --filter @workspace/feiticeiros run dev` вҖ” run the frontend (port auto, via workflow)
- `pnpm run typecheck` вҖ” full typecheck across all packages
- `pnpm run build` вҖ” typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` вҖ” regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` вҖ” push DB schema changes (dev only)
- Required env: `DATABASE_URL`, `CLERK_SECRET_KEY`, `CLERK_PUBLISHABLE_KEY`, `VITE_CLERK_PUBLISHABLE_KEY`

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5 (`artifacts/api-server`)
- Frontend: React + Vite + Tailwind v4 + shadcn/ui (`artifacts/feiticeiros`)
- DB: PostgreSQL + Drizzle ORM (`lib/db`)
- Auth: Replit-managed Clerk (`@clerk/express` server, `@clerk/react` frontend)
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec at `lib/api-spec/openapi.yaml`)
- Router: wouter v3
- State: TanStack Query v5

## Where things live

- `lib/api-spec/openapi.yaml` вҖ” source-of-truth API contract

## Architecture decisions

- Contract-first API: OpenAPI spec вҶ’ Orval codegen вҶ’ typed hooks + Zod validators used across frontend and backend
- Characters are user-scoped (requireAuth + userId filtering on all character routes)
- Clerk auth uses proxy middleware вҖ” `publishableKeyFromHost` from `@clerk/react/internal` on frontend, `@clerk/express` on backend
- All text fields (backstory, abilities, equipment, etc.) stored as JSON strings in text columns for flexibility
- Auto-save pattern on character-sheet with 900ms debounce and revision tracking

## Gotchas

- `@clerk/express@2.x` requires `@clerk/shared@^4.x` вҖ” keep both in sync
- `@clerk/react@6.x` requires `@clerk/shared@^4.x` вҖ” must be in `minimumReleaseAgeExclude` in pnpm-workspace.yaml since these are recent releases
- After any OpenAPI change, run `pnpm --filter @workspace/api-spec run codegen` before building
- Clerk warning "loaded with development keys" is EXPECTED in dev вҖ” not a bug
- API routes at `/api/*` are protected by requireAuth; curl tests will return 401 without a session cookie
- `publishableKeyFromHost` is imported from `@clerk/react/internal` on the frontend (NOT from `@clerk/shared/keys` directly)
